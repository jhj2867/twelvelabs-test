🎯 접근 방식

모든 테스트는 실제 TwelveLabs API를 직접 호출하는 통합 테스트입니다. 목(mock)은 사용하지 않았습니다.

  • SDK가 API 요청을 올바르게 직렬화하는지 검증
  • API 응답이 SDK 타입(SearchResults, SearchItem 등)으로 올바르게 역직렬화되는지 검증
  • 실제 엔드포인트에서의 에러 응답 코드와 에러 타입 매핑 검증
  • SDK 문서와 실제 동작의 차이 발견 (예: query() vs create())

제외한 항목

  • query_media_url / query_media_file / query_media_type — 공개 이미지 URL 필요, 이번 범위 제외
  • sort_option — 공식 문서(1.3)에 명시되어 있으나 실제 SDK 1.2.2에 존재하지 않아 제외
  • 인덱스 생성 및 영상 업로드 — 사전 조건으로 간주

⚠️ 가정 사항

  1. 사전 인덱싱된 영상 — 테스트 실행 전 인덱스에 최소 1개의 영상이 업로드·인덱싱되어 있어야 합니다.
  2. 인증 우선 검증 — API는 파라미터 유효성 검사 전에 인증을 먼저 수행합니다. 에러 처리 테스트에도 유효한 API 키가 필요합니다.
  3. 실시간 API 의존성 — 네트워크 상태나 TwelveLabs 서비스 상태에 따라 결과가 영향을 받을 수 있습니다.
  4. 페이지 토큰 유효기간 — 토큰은 일정 시간 후 만료됩니다. PA-06은 만료 토큰 에러를 검증합니다.
